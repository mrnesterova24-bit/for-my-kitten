# Разместить сайт на Vercel (без своего сервера)

Подходит, если у тебя **виртуальный хостинг** reg.ru без доступа root. Vercel бесплатно соберёт проект из GitHub и даст ссылку на сайт. Домен for-my-kitten.ru привяжешь в Vercel и в reg.ru.

---

## Шаг 1. Регистрация и проект на Vercel

1. Открой **https://vercel.com** и нажми **Sign Up**. Войди через **GitHub** (тот же аккаунт, где репозиторий for-my-kitten).
2. После входа нажми **Add New…** → **Project**.
3. Найди репозиторий **mrnesterova24-bit/for-my-kitten** и нажми **Import**.
4. Оставь настройки по умолчанию (Framework: Next.js, Root Directory: .) и нажми **Deploy**.
5. Подожди 1–2 минуты. Когда появится зелёная галочка — сайт собран. Будет ссылка вида **https://for-my-kitten-xxx.vercel.app**. Открой её и проверь, что сайт открывается.

---

## Шаг 2. Привязать свой домен for-my-kitten.ru

1. В проекте на Vercel открой вкладку **Settings** → **Domains**.
2. В поле введи **for-my-kitten.ru** и нажми **Add**. Добавь также **www.for-my-kitten.ru**.
3. Vercel покажет, какие записи нужны в DNS. Обычно это:
   - для **for-my-kitten.ru** — запись **A** с именем **@** на адрес **76.76.21.21** (или CNAME на **cname.vercel-dns.com** — смотри, что именно напишет Vercel);
   - для **www** — запись **CNAME** с именем **www** на **cname.vercel-dns.com**.

---

## Шаг 3. Настроить DNS в reg.ru

1. Зайди в **reg.ru** → домен **for-my-kitten.ru** → **Зона DNS** (управление записями).
2. Сделай записи **точно так**, как написал Vercel в шаге 2. Обычно:
   - Тип **A**, имя **@**, значение **76.76.21.21** (или тот IP, что указан у Vercel).
   - Тип **CNAME**, имя **www**, значение **cname.vercel-dns.com**.
3. Сохрани. Подожди 5–30 минут — DNS обновится.

После этого **https://for-my-kitten.ru** будет открывать твой сайт на Vercel. HTTPS Vercel включит сам.

---

## Важно: данные и загрузки

На Vercel приложение работает в «облаке» без твоего диска. Файлы в **src/data/*.json** и **public/uploads** хранятся в репозитории: что есть в GitHub при деплое — то и будет на сайте. Чтобы менять контент (мемы, секретная комната, письма и т.д.), нужно либо:
- править JSON в репозитории и пушить в GitHub (после каждого пуша Vercel пересоберёт сайт),  
либо позже подключить базу данных или файловое хранилище (это отдельная настройка).

Для начала достаточно пуша в GitHub: правки в коде и контенте → коммит → push → через минуту сайт на Vercel обновится.
