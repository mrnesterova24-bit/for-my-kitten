# Начни здесь — выложить сайт за 3 шага

**Если ты заходишь на сервер по SSH как пользователь u3415733 и при `sudo` пишет «права суперпользователя отсутствуют»** — это **виртуальный хостинг**, а не VPS. Инструкция ниже (скрипт, Nginx, certbot) для **VPS с root**. В твоём случае проще разместить сайт на **Vercel** без своего сервера — открой файл **VERCEL.md** и делай по нему.

---

Тебе нужно сделать только **3 вещи**. Каждое — один раз. *(Ниже — для VPS, где есть root/sudo.)*

---

## Сразу после покупки VPS

1. Зайди в личный кабинет reg.ru → раздел с **VPS/облачный сервер**.
2. Найди свой новый сервер и посмотри **IP-адрес** (например, 123.45.67.89) и **пароль root** (часто приходит отдельно по почте или в панели «Пароль» / «Доступ»).
3. В этой инструкции везде, где указан адрес **31.31.197.5**, подставляй **свой IP** вместо него.

Дальше — шаги 1–3 ниже.

---

## Шаг 1. Домен (reg.ru)

1. Открой в браузере: **https://www.reg.ru**
2. Войди в свой аккаунт.
3. Найди домен **for-my-kitten.ru** и нажми «Управление» или «Зона DNS».
4. Найди таблицу с записями (A, CNAME и т.д.). Нужны две записи типа **A**:
   - Имя **@** (или пусто) → значение **31.31.197.5**
   - Имя **www** → значение **31.31.197.5**
   Если таких записей нет — добавь. Если есть, но с другим числом — замени на **31.31.197.5**.
5. Сохрани.

Готово. Через 5–30 минут домен начнёт открываться на твоём сервере.

---

## Шаг 2. Сервер (один раз запустить скрипт)

Тебе нужен доступ к серверу **31.31.197.5** (логин и пароль тебе дал хостинг).

### Как подключиться к серверу

- **Windows:** открой «Командную строку» или PowerShell и введи:
  ```
  ssh root@31.31.197.5
  ```
  Или скачай программу **PuTTY**, в поле «Host» введи **31.31.197.5**, нажми «Open», затем введи логин (часто **root**) и пароль.

- Когда увидишь приглашение вроде `root@сервер:~#` — ты на сервере.

### Что ввести на сервере

**Способ 1 (если сработает).** Вставь команду и нажми Enter:

```bash
curl -sSL https://raw.githubusercontent.com/mrnesterova24-bit/for-my-kitten/main/scripts/server-setup.sh | sudo bash
```

Если появится **404** или «команда не найдена» — переходи к способу 2.

**Способ 2 (вставить скрипт вручную).**

1. На **своём компьютере** открой в этом проекте файл **scripts/server-setup.sh**. Выдели весь текст (Ctrl+A), скопируй (Ctrl+C).

2. На **сервере** в PuTTY введи команду и нажми Enter:
   ```bash
   nano setup.sh
   ```

3. В открывшемся редакторе **вставь скрипт** (в PuTTY — правый клик мыши). Проверь, что вставилось с первой по последнюю строку.

4. Сохрани и выйди: нажми **Ctrl+O**, затем **Enter**, затем **Ctrl+X**.

5. Запусти скрипт (нужны права администратора):
   ```bash
   sudo bash setup.sh
   ```
   Если попросит пароль — введи пароль своего пользователя на сервере (при вводе символы не отображаются, просто введи и нажми Enter).

Скрипт установит Node.js, Nginx, скачает проект, соберёт и запустит его. Подожди 3–5 минут. В конце должно появиться «Готово».

## Шаг 3. HTTPS (один раз, когда сайт уже открывается по http)

Когда в браузере открывается **http://for-my-kitten.ru** (даже без «замочка»), подключи бесплатный HTTPS:

1. Снова подключись к серверу: `ssh root@31.31.197.5`
2. Введи одну команду:
   ```bash
   certbot --nginx -d for-my-kitten.ru -d www.for-my-kitten.ru
   ```
3. Введи свой email и согласись с условиями (Y). Certbot сам настроит «замочек».
4. Готово. Теперь сайт открывается по **https://for-my-kitten.ru**.

## Если что-то пошло не так

- **Не могу зайти на сервер** — проверь логин/пароль у хостинга (reg.ru или тот, кто дал 31.31.197.5). Часто логин **root**, пароль приходит в письме при заказе VPS.
- **Сайт не открывается** — подожди до 30 минут после шага 1 (DNS обновляется). Проверь: в командной строке на своём компьютере введи `ping for-my-kitten.ru` — должен отвечать 31.31.197.5.
- **Ошибка при запуске скрипта** — пришли текст ошибки, разберём.
